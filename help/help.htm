Also_XH - a CMSimple_XH Plugin

Also_XH facilitates to have an arbitrary amount of additional contents on your website. These are similar to newsboxes, but instead have different content for every page. Also_XH is inspired by the Also plugin, but allows editing with the chosen editor instead of a textarea (CMSimple_XH ≥ 1.5 only).

Requirements

Also_XH is a plugin for CMSimple_XH ≥ 1.2. It requires a UTF-8 encoded version.

Installation

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory also/ to your server into CMSimple_XH's plugins directory.
  4. Set write permissions to the subdirectories config/, css/, languages/ and the plugin's data folder.
  5. Switch to "Also" in the back-end to check if all requirements are fulfilled.

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select "Also" from "Plugins".

You can change the default settings of Also_XH in "Plugin Config". Hints for the options will be displayed when hovering over the help icon with your mouse.

Localization is done in "Plugin Language". You can translate the character strings to your own language, or customize them according to your needs.

The look of the Also_XH can be customized in "Plugin Stylesheet".

Usage

Also_XH is used from the template. To be able to edit the additional content with an editor, you have to insert <?php echo also_enable();?> before <?php echo head();?>

To have additional content on your site, just insert <?php echo also('my_content');> in the place where you want, e.g. instead of a newsbox. Instead of 'my_content' you can choose any name, as long as it consists of lower-case letters (a-z), digits and underscores only. You can have as many additional contents as you like, as long as you give them different names.

If you are in edit mode, you can edit the additional content, either with a textarea or from CMSimple_XH 1.5 on with the installed editor. Note that the tinyMCE shipped with CMSimple_XH 1.5 and 1.5.1 doesn't work with Also_XH, so currently only CKEditor can be used. If you want to use another toolbar than the one you have configured for the main content editor, just give the name of the toolbar as second parameter, e.g. <?php echo also('small_content', 'sidebar');>.

The additional content is stored in Also_XH's data folder in an (X)HTML file with the name given as first parameter, e.g. my_content.htm. The structure is similar to that of CMSimple's content. The pages are seperated by <h1>, <h2> or <h3> according to their menu level. The name of the page headings are inserted for better readability only; they are ignored by Also_XH. Instead the link to the pages is made with the id given for the headings. You should not alter these id's in any way! Inserting new pages by inserting new headings is not possible; these will simply be ignored.
